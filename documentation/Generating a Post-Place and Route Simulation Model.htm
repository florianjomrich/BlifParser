Generating a Post-Place and Route Simulation Model
You can generate a simulation model after placing and routing your design. For this process, NetGen converts the Place & Route process results (an NCD file) to the simulation model (a structural SIMPRIM-based VHDL or Verilog file) and a standard delay format (SDF) file. The SDF file contains true timing delay information for your design.
Note Running this process is equivalent to running the Place and Route process with the Generate Post-Place & Route Simulation Model property set.
The simulation model file and SDF file can be used to verify the functionality and timing of your design at this point in the design flow by using the Simulate Post-Place and Route Model process.  
Note For simulators not integrated with ISE®, see the simulator documentation for information on simulating your placed and routed design.
Run Place & Route on the top module .  
To Generate a Post-Place & Route Simulation Model
  1.  In the Processes tab, expand Place & Route.
  2.  Right-click Generate Post-Place & Route Simulation Model, and select Properties from the right-click menu.  
  3.  Set the desired property values in the Simulation Model Properties, such as the simulation model file type (VHDL or Verilog) in the Simulator Model Target property.
    Note If you ran Place & Route with the Generate Post-Place & Route Static Timing property set to Yes (default setting), you have already generated the post-place and route static timing report. The status will appear as complete (green checkbox).
  4.  Double-click the Generate a Post-Place & Route Simulation Model process.
  •  NetGen generates a simulation model (a VHDL or Verilog file) and an SDF file from the Place & Route process results.  For more information about NetGen, see the Development System Reference Guide.
    The post-place and route simulation model is named <top_level_design_name>_timesim.v (or .vhd).
    Note The simulation model file type (VHDL or Verilog) is determined by the Simulator Model Target property in the Simulation Model Properties.
  •  The simulation model generated by this process can be used as an input file for your simulator.
Run the Simulate Post-Place & Route Model process.
Note For simulators not integrated with ISE, see the documentation for your simulator for information on simulating your placed and routed design.
